# Trade Logging System - Complete Reference

## 📊 Trade Data Persistence

All trades are automatically saved with **multiple layers of protection** to prevent data loss:

### 1. Automatic Export Triggers

| Trigger | Frequency | File Location | Prefix |
|---------|-----------|---------------|--------|
| **Per-trade** | Every 5 trades | `trades/` | `live_t{count}_trades_{timestamp}.csv` |
| **Periodic auto-save** | Every 50 bars | `trades/` | `autosave_b{bar_count}_trades_{timestamp}.csv` |
| **Emergency shutdown** | On Ctrl+C / SIGTERM | `trades/` | `shutdown_{timestamp}_trades_{timestamp}.csv` |

### 2. Export File Types

Each export creates **2 files**:

1. **`*_trades_{timestamp}.csv`** - Detailed trade-by-trade data
   - 28 columns including: trade_num, entry/exit times, prices, PnL, MFE/MAE, capture efficiency, dual-agent metrics
   
2. **`*_summary_{timestamp}.csv`** - Performance summary
   - Overall metrics: total trades, win rate, profit factor, Sharpe, Sortino, max drawdown, etc.

### 3. Data Columns (Trade CSV)

```
trade_num, entry_time, exit_time, duration_seconds, direction,
entry_price, exit_price, price_change, pnl, pnl_percent,
mfe, mae, mfe_percent, mae_percent, capture_efficiency,
winner_to_loser, equity_after, result,
predicted_runway, runway_utilization, runway_error_pct,
trigger_quality, harvester_quality, mfe_bar_offset,
mae_bar_offset, bars_from_mfe_to_exit
```

## 🛡️ Data Safety Features

### 1. Multiple Redundancy Layers

- ✅ **Every 5 trades**: Immediate export after trade closes
- ✅ **Every 50 bars**: Periodic auto-save (even if no trades)
- ✅ **On shutdown**: Emergency export before bot stops
- ✅ **Duplicate prevention**: Tracks `last_export_count` to avoid re-exporting same trades

### 2. Error Handling

All export operations are wrapped in try/except:
- Failures logged with full traceback
- Doesn't crash bot if export fails
- Continues trading even if CSV write fails

### 3. Timestamp Precision

All exports use microsecond-precision UTC timestamps:
```
YYYYMMDD_HHMMSS format (20260110_143022)
```

### After 500+ Trades (Graduation Check)

```bash
# 1. Find most recent complete export
LATEST_CSV=$(ls -t trades/live_t*_trades_*.csv | head -1)

# 2. Run full analysis
python3 trade_analyzer.py "$LATEST_CSV"

# 3. Export to JSON for archival
python3 trade_analyzer.py "$LATEST_CSV" --export "phase1_graduation_$(date +%Y%m%d).json"

# 4. Check graduation criteria
python3 -c "
import pandas as pd
import numpy as np

df = pd.read_csv('$LATEST_CSV')
returns = df['pnl'].values
sharpe = (returns.mean() / (returns.std() + 1e-8)) * np.sqrt(252)
downside = returns[returns < 0].std()
sortino = (returns.mean() / (downside + 1e-8)) * np.sqrt(252)
win_rate = (df['pnl'] > 0).mean()

print(f'Sharpe: {sharpe:.2f} (need > 1.0)')
print(f'Sortino: {sortino:.2f} (need > 0.8)')
print(f'Win Rate: {win_rate*100:.1f}% (need ≥ 45%)')
print(f'Trades: {len(df)} (need 500+)')

if sharpe > 1.0 and sortino > 0.8 and win_rate >= 0.45 and len(df) >= 500:
    print('\n✅ READY FOR PHASE 2')
else:
    print('\n❌ CONTINUE PHASE 1')
"
```

## 🔧 Troubleshooting

### "No trades CSV files found"

**Cause:** Bot hasn't completed 5 trades yet or trades/ directory empty

**Fix:**
```bash
# Check if bot is running
ps aux | grep ctrader_ddqn_paper.py

# Check total trade count in logs
grep "Total trades:" logs/live_micro/learning_*.log | tail -1

# Manually check PerformanceTracker state
grep "PERF" logs/live_micro/learning_*.log | tail -10
```

### "CSV export failed"

**Cause:** Disk full, permissions issue, or corrupted PerformanceTracker

**Fix:**
```bash
# Check disk space
df -h .

# Check permissions
ls -ld trades/

# Create if missing
mkdir -p trades && chmod 755 trades

# Check recent errors
grep "EXPORT.*Failed" logs/live_micro/learning_*.log
```

### "Analysis shows 0 trades"

**Cause:** CSV file corrupted or only has headers

**Fix:**
```bash
# Check file size
ls -lh trades/*.csv

# Count rows
wc -l trades/*.csv

# Inspect first 10 lines
head -10 trades/latest.csv

# If corrupted, look for backup
ls -t trades/*autosave*.csv
```

## 📋 Best Practices

### 1. Regular Backups

```bash
# Daily backup script (add to cron)
#!/bin/bash
BACKUP_DIR="/backup/ctrader_trades/$(date +%Y%m%d)"
mkdir -p "$BACKUP_DIR"
cp -r trades/*.csv "$BACKUP_DIR/"
echo "Backed up $(ls trades/*.csv | wc -l) files to $BACKUP_DIR"
```

### 2. Archival After Phase Completion

```bash
# After Phase 1 graduation
tar -czf phase1_complete_$(date +%Y%m%d).tar.gz \
    trades/ \
    logs/live_micro/ \
    analysis_*.json

mv phase1_complete_*.tar.gz /archive/
```

### 3. Data Validation

```bash
# Check for duplicate trade_num
python3 -c "
import pandas as pd
df = pd.read_csv('trades/latest.csv')
duplicates = df[df.duplicated('trade_num', keep=False)]
if len(duplicates) > 0:
    print('⚠ Found duplicate trade numbers:')
    print(duplicates[['trade_num', 'entry_time']])
else:
    print('✓ No duplicates')
"
```
